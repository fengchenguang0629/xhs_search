# 小红书笔记搜索与摘要

## 快速开始

### 使用示例

用户输入搜索需求时：

```
从小红书搜索"产后修复"的相关笔记并总结
```

### 处理流程

此技能使用以下工作流程：

1. **搜索阶段**：调用本地Flask API搜索小红书笔记
2. **图片提取**：自动下载笔记中的图片并转换为Base64格式
3. **数据准备**：组织笔记信息和图片分析任务
4. **模型处理**：当前模型（Claude、GPT等）处理：
   - 分析图片内容（分批处理，每批5张，防止上下文溢出）
   - 融合图片信息与笔记文字内容
   - 生成带引用的总结

## 核心特性

### 无三方客户端

- **不依赖Anthropic SDK**：直接使用调用技能的模型能力
- **图片Vision处理**：由当前模型的Vision API完成
- **灵活适配**：支持任何兼容的LLM（Claude、GPT等）

### 分批图片处理

- 每批处理5张图片，防止单次请求Token过多
- 自动过滤无用图片（返回"无用"的内容）
- 融合多张图片的有用信息

### 标准化输出格式

- **自然语言总结**：清晰易读的Markdown格式
- **上标引用**：使用 `[1]`、`[2]` 标记笔记来源
- **参考资料**：文末附加紧凑的链接列表

## 工作流程详解

### 步骤1：调用搜索API

调用搜索API获取笔记和图片：

```
from xhs_notes_search.scripts.summarizer import search_and_prepare

data = search_and_prepare(
    query="二月龄小宝宝喂养指南",
    require_num=10,
    analyze_images=True
)
```

返回的数据结构：
```json
{
  "status": "success",
  "query": "搜索词",
  "notes": [
    {
      "index": 1,
      "title": "笔记标题",
      "content": "笔记内容...",
      "url": "https://...",
      "images": [
        {
          "base64": "...",
          "media_type": "image/webp",
          "url": "原始图片链接"
        }
      ]
    }
  ],
  "image_analysis_tasks": [
    {
      "note_index": 1,
      "note_title": "...",
      "batches": [
        {

            {
              "base64": "...",
              "media_type": "image/webp"
            }
          ]
        }
      ],
      "prompt": "请分析这些图片..."
    }
  ],
  "prompt_template": "请根据以下笔记生成总结..."
}
```

### 步骤2：模型处理图片分析

对每个 `image_analysis_tasks` 项，使用Vision API分析：

- 迭代所有图片任务
- 对每个batch中的图片调用Vision分析
- 收集分析结果
- 自动过滤标记为"无用"的内容

### 步骤3：生成总结

使用模型将图片分析结果和笔记内容进行融合：

- 将图片分析结果插入提示词
- 调用LLM生成最终摘要
- 使用Markdown上标引用格式
- 生成参考资料列表

## 配置参数

### search_and_prepare() 参数

- **query** (str, 必需)：搜索关键词
- **require_num** (int, 可选，默认10)：返回笔记数量
- **analyze_images** (bool, 可选，默认True)：是否包含图片分析任务
- **api_url** (str, 可选)：本地Flask API地址（默认 http://localhost:5000）

### 高级选项

如用户要求"详细"或"深度"分析，可调整参数：

- 增加 `require_num` 参数（例如20而不是10）获得更全面的信息
- 确保 `analyze_images=True` 以包含图片分析
- 对于多个搜索维度，可多次调用搜索API

## 技术细节

### 图片处理

1. **下载**：从笔记中提取图片URL并下载
2. **编码**：转换为Base64格式供模型识别
3. **分批**：每批5张，避免单次请求过大
4. **过滤**：模型标记"无用"的图片会被自动排除
5. **融合**：有用信息整合到最终总结中

### 错误处理

- 搜索API不可用：返回 `{'status': 'error', 'message': '...'}`
- 无结果：返回 `{'status': 'no_results', 'message': '...'}`
- 图片分析失败：该笔记的图片信息被跳过，但不影响总体总结

## 故障排除

### API连接失败
检查本地Flask服务是否运行：
```bash
python web/app.py
```

### 图片分析效果差
- 确保模型支持Vision API
- 图片内容清晰且相关
- 检查网络连接以确保图片正确下载

### Token消耗过多
- 系统已设置每批5张图片的限制
- 可调整 `summarizer.py` 中的 `batch_size` 参数
- 减少 `require_num` 参数可降低总体内容量
